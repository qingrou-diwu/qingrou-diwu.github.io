无符号数和有符号数

无符号数

寄存器的位数反映无符号数的表示范围

8位 ：0 ~ 255
16位 ：0 ~ 65535

有符号数

  1. 机器数与真值：

    真值 机器数
    带符号的数 符号数字化的数
    +0.1011 0.1011
    -0.1011 1.1011
    +1100 0,1100
    -1100 1,1100

  2. 原码表示法：整数直接在最高位前添加符号位,逗号隔开，小数将小数点前的一位作为符号位，正数为0，负数为1。

    真值 原码
    +0.1011 0.1011
    -0.1011 1.1011
    +1100 0,1100
    -1100 1,1100
    +0.0000 0.0000
    -0.0000 1.0000
    +0 0,0000
    -0 1,0000

  3. 反码表示法：正数的反码是本身，负数的反码是除符号位以外取反

    真值 原码 反码
    +0.1011 0.1011 0.1011
    -0.1011 1.1011 1.0100
    +1100 0,1100 0,1100
    -1100 1,1100 1,0011
    +0.0000 0.0000 0.0000
    -0.0000 1.0000 1.1111
    +0 0,0000 0,0000
    -0 1,0000 1,1111

  4. 补码表示法：符号位不变，反码+1

    真值 原码 反码 补码
    +0.1011 0.1011 0.1011 0.1011
    -0.1011 1.1011 1.0100 1.0101
    +1100 0,1100 0,1100 0,1100
    -1100 1,1100 1,0011 1,0100
    +0.0000 0.0000 0.0000 0.0000
    -0.0000 1.0000 1.1111 0.0000
    +0 0,0000 0,0000 0,0000
    -0 1,0000 1,1111 0.0000

  5. 移码表示法：补码符号位取反

数的定点表示和浮点表示

定点表示

  1. 概念：小数点按约定方式标出
  2. 小数：数符+小数点+数值（小数定点机）0 ≤ |x| ≤ ( 1 - 1/2ⁿ )
  3. 整数：数符+数值+小数点（整数定点机）0 ≤ |x| ≤ ( 2ⁿ - 1 )

浮点表示

浮点数的表示形式

  1. 一般形式： N = S × rⁿ （ 正确公式中n应为 j ）
  2. N：尾数 n：阶码 r：尾数的基值 （ r 一般为2,4,8,16 ）

浮点数的表示范围

上溢 | 负数区 | 下溢 | 正数区 | 上溢
———————————————————————————————————————————
最小负数 最大负数 最小正数 最大正数

上溢：阶码＞最大阶码
下溢：阶码＜最小阶码 按机器零处理

浮点数的规格化

阶符+阶码+数符+尾数

例1：将19/128写成二进制定点数、浮点数及在定点机和浮点机中的机器数形式。其中数值部分均取 10 位，数符取 1 位，浮点数阶码取 5 位（含1位阶符），尾数规格化。

二进制形式 x = 0.0010011
定点表示 x = 0.0010011000
浮点规格化表示 x = 0.1001100000 × 2^-10
定点表示 x = 0.0010011000
浮点表示 x = 1,0010;0.1001100000

例2：将-58写成二进制定点数、浮点数及在定点机和浮点机中的机器数形式。其中数值部分均取 10 位，数符取 1 位，浮点数阶码取 5 位（含1位阶符），尾数规格化。

二进制形式 x = -111010
定点表示 x = -0000111010
浮点规格化表示 x = -（0.1110100000）×2^110
定点表示 x = 1,0000111010
浮点表示 x = 0,0110;1.1110100000

IEEE 754 标准

32位浮点数：第1位为数符位，中间8位为阶码+127，即+01111111，后23位为尾数

754标准32位二进制转十进制：根据第一位判断正负，中间8位-127即为阶数，真实尾数为尾数作为小数部分加1。

例1：（41360000）16，转换为十进制

  • 展开为二进制0100 0001 0011 0110 0000 0000 0000 0000
  • 数符为0:意为是正数
  • 阶码为10000010，减去偏移值01111111（127) 后为3
  • 尾数为011011，作为小数部分加1后为1.011011
  • x = 1.011011 × 2³ = 1011.011 = 11.375

十进制转754标准32位二进制：转化为二进制，移小数点到第一位后去掉首位即为尾数，移动小数点的位数+127转化为二进制即为阶码，根据正负添数符。

例2：（20.59375）10 转化为754标准的32位浮点数二进制

  • 转化为二进制：10100.10011
  • 移小数点到第一位后去掉1即为尾数：010010011
  • 小数点移动了4位，则阶码为4+127=1000+01111111=00000100+10000000-1=10000011
  • 由于是正数，数符为0
  • 二进制格式为：0100 0001 1010 0100 1100 0000 0000 0000
  • 十六进制格式为：41A4C000

64位浮点数：第1位为数符位，中间11位为阶码+1023，即+01111100011，后52位为尾数
